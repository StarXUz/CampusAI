## 一、当前已经较完整覆盖的部分

1. 多模块 Maven 结构
   已具备 `common`、`service`、`web` 三层拆分，符合课程要求的模块化设计。

2. 三端角色系统
   已形成学生端、后台管理端、超级管理端三端协同，并通过不同 host + port 隔离登录态。

3. 权限与登录
   已具备 Spring Security 登录、退出、记住我、角色跳转、跨端重定向修正和 API 鉴权错误统一处理。

4. 校园餐饮业务
   已具备商家、分类、菜品、套餐、图片上传、上架 / 售罄、学生端点餐展示与订单联动。

5. 自习室预约
   已支持自习室 CRUD、预约规则、座位状态展示、预约记录查看、模板下载、Excel 导入、FullCalendar 视图和基础并发防护。

6. 活动报名与静态化
   已具备活动 CRUD、学生报名、取消报名、PDF 报表导出、静态页生成、过期自动刷新和 Nginx 静态说明。

7. AI 助手
   已接入兼容 OpenAI 协议的千问接口，并能结合数据库做问答演示。

8. 后台统计页
   已落地 ECharts 图表统计，不再是纯文本或静态卡片页。

9. 报表导出
   已具备月度运营 Excel 导出与 PDF 导出，且与页面预览共用同一份数据库统计结果。

10. 审计日志
    已覆盖商家、菜品、套餐、自习室、活动等关键后台操作。

## 二、当前可作为加分与演示亮点的部分

1. Redis 缓存
   已接入验证码缓存、热点数据缓存、图片缓存指标展示，可作为缓存能力演示点。

2. Quartz / 定时任务
   已具备未支付订单取消、活动过期刷新、孤儿图片清理等任务，可作为调度能力展示点。

3. 活动静态页
   已支持生成 HTML、自动刷新、生成后直接打开，可完整体现“动态转静态”的实现链路。

4. 超级管理端
   已支持学生账号与后台账号治理，并与其他端共享数据库联动。

5. 多端隔离
   三端 host 拆分后已解决 Cookie 串端问题，能够直观体现多端登录隔离能力。

## 三、当前仍可进一步完善的部分

1. ZooKeeper 业务化程度一般
   当前已保留配置中心读取与运行状态展示，但默认未启用，主要体现可扩展能力，不属于核心业务依赖。

2. AI 高分功能仍可补强
   当前 AI 已能基于数据库完成问答，并预留菜品推荐、活动摘要、预约建议、饮食分析等能力扩展空间。

3. 商家管理员权限可再细化
   当前商家范围已做基础收口，订单与报表粒度仍有进一步细化空间。

4. 交付材料仍需完善
   项目代码与功能已较完整，仍需补齐部署说明、演示脚本、架构图、流程图等提交材料。

## 四、当前需要重点避免的失分点

1. 文档和代码状态不一致
   之前已有部分文档停留在旧状态，现在应以当前代码和当前演示路径为准。

2. 课堂环境准备不充分
   三端地址、默认账号、数据库配置、Redis 是否可连、桌面模板文件等需要提前准备好。

3. AI 和 ZooKeeper 说明不清
   答辩口径：
   - AI 当前以数据库问答与智能助手为主
   - ZooKeeper 当前未启用，不影响主体业务，属于扩展型协调配置能力展示

4. 材料缺少“同源性”说明
   答辩口径：
   - 月报预览 / Excel / PDF 同源
   - 活动报名人数 / 静态页 / PDF 同源
   - 三端共享同一数据库
